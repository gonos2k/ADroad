## 종합 판정
방향(JAX·Python 기반·변분 동화)은 타당하고 문서 구조도 좋다. 그러나 계획서는 **"어려운 부분"을 체계적으로 과소평가**한다. 반복되는 낙관("implicit solver가 없어 크게 유리")이 실제 난점(이산 저장항 로직의 gradient 병리, CFL 안정성, 식별성, 상변화 위치 혼동)을 가린다. 아래 **Critical 6건은 착수 전에 계획을 고쳐야** 하는 항목이다. 특히 C1(상변화 위치 혼동), C2(CFL×보정 상호작용), C3(식별성/데이터 부족), C4(무엇을 이겨야 성공인가)이 핵심이다.

## Critical — 착수 전 수정 필요

### C1. 엔탈피법이 "엉뚱한 방정식"을 고칠 위험 (물리 오류)
계획서는 동결 시 노면온도 진동(N10)을 **엔탈피(유효 열용량)법**으로 해결한다고 한다. 그러나 RoadSurf에는 상변화가 **두 곳**에 있고 성격이 다르다:
- **(a) 지중 온도 방정식**: 동결 시 잠열이 지층 온도에 반영되지 않음 → 논문 3.6이 말하는 "노면온도가 동결한계 위아래로 진동"은 **여기서** 발생. 유효 열용량법이 적용되는 곳.
- **(b) 표면 저장항**: 물(mm) → 얼음(mm)의 **질량 이전**(식 46–50). 이것은 단일 매질의 온도-엔탈피가 아니라 **별개 저수지 간 질량 전달**이라 유효 열용량법으로 매끄럽게 만들 수 없다. 여전히 게이트/질량보존 재정식화가 필요.

계획서는 이 둘을 뭉뚱그렸다. 엔탈피법만 넣으면 (b)의 이산 전환·질량 이전은 그대로 남아 gradient 병리와 질량 비보존이 지속된다.
- **대응**: §5를 (a)/(b)로 분리. (a)=지층 엔탈피법, (b)=**질량보존형 soft 전달**(녹은분율 φ로 두 저수지 사이 질량을 연속 분배하되 $\sum$ 저장항 보존을 명시적 제약으로). 두 처리의 상호작용(융해수가 지층 온도에 주는 피드백)도 검증 대상에 추가.

### C2. 명시적 스킴 CFL과 파라미터 보정의 충돌 (수치 안정성)
계획서는 CFL을 "주의"로만 언급한다. 그러나 이는 보정과 정면충돌한다:
- 표층이 얇고(~1–2 cm) Δt=30 s. 안정조건 대략 $\Delta t < \Delta z^2/(2\alpha)$. 아스팔트 열확산율에서 30 s는 **경계선**이다.
- **보정 대상 파라미터(열전도 $K$, 열용량 $\rho_g c_g$, 공극률)가 바로 $\alpha$를 바꾼다.** 최적화 도중 gradient step이 $K$를 키우면 스킴이 **발산 → NaN → gradient NaN**. 즉 손실 지형에 "폭발 절벽"이 생겨 최적화가 깨진다.
- **대응**: (i) 안정성 여유를 확보하도록 표층 Δt에 **서브스텝** 도입 또는 상단 몇 층만 **암시적(θ-method)** 처리(이 경우 미분가능 선형 solve 필요 — "implicit 없음" 장점 일부 포기), (ii) 파라미터 재매개화 시 CFL 여유를 제약으로 반영, (iii) 발산 감지 시 손실에 페널티/clip. DoD에 "보정 전 구간에서 CFL 만족" 체크 추가.

### C3. 식별성·데이터 충분성 — 최우선 과학 리스크인데 각주 취급
목적이 "파라미터 개선"인데, 계획서는 식별성을 리스크 표의 한 줄로만 다룬다. 실제로는:
- 파라미터 ~70개(물리 30 + 노면 40). 다수가 **임계값**(TLimFreeze, TLimMelt…)이라 **특정 기상에서만 활성**. 강수·상변화가 없는 구간에서는 저장항 파라미터 gradient가 **정확히 0**.
- 검증 데이터가 **단일 지점·단일 사례**(`example_data/test_input.csv`, 108 h) 하나. 이걸로 보정하면 **과적합·비유일(equifinality)** 이 거의 확실.
- **대응**: (i) 계획을 "단일 파일 데모"가 아니라 **다중 사례·다중 지점 데이터 확보**를 M0의 필수 산출물로 격상. (ii) 보정 대상을 민감도 스크리닝(Morris/Sobol 또는 gradient-norm 랭킹)으로 **3–5개로 축소**하고 나머지는 고정. (iii) 정칙화·사전분포·범위제약을 필수화. (iv) 쌍둥이 실험을 "옵션(M6)"이 아니라 **식별성 게이트**로 앞당김.

### C4. "무엇을 이겨야 성공인가"가 없음 (검증 가능성)
DoD 3번은 "데모가 수렴"이다. 적대적으로 말하면: **모든 마일스톤을 끝내고도 아무것도 입증하지 못할 수 있다.**
- 노면온도 관측만 쓴다면, 기존 **커플링이 이미 그 일을 한다**(복사보정으로 Tsurf 정합). "초기상태 gradient 동화"가 커플링 대비 **새로 주는 가치**가 명시돼 있지 않다.
- **대응**: 정량 성공기준을 못박기 — 예: (a) **쌍둥이 실험에서 알려진 파라미터를 X% 이내 복원**, (b) 독립 검증구간에서 **운영 커플링 대비 노면온도 RMSE 동등 이상**, (c) 다중 사례에서 보정 파라미터가 물리적으로 타당·안정. "커플링을 이기거나 대체할 근거"를 프로젝트 가설로 1페이지 추가.

### C5. 검증 허용오차 1e-6 °C는 비현실적 (branch flip의 카오스성)
`exact` 모드에서 Python 대비 노면온도 RMSE < 1e-6 요구는:
- 저장항/상변화가 활성인 구간에서는 **임계값 근처 branch flip** 한 번이면 궤적이 임의로 갈라진다(임계점 부근 사실상 카오스적). 108 h 롤아웃에서 1e-6은 **무강수·무상변화** 경우에만 성립할 것.
- XLA의 부동소수점 재결합·`fma`·연산 순서가 numpy와 달라, branch가 없어도 1e-6은 낙관적.
- **대응**: 허용오차를 **레짐별**로 재정의 — 순수 열전도 구간 <1e-9(상대), 상변화 활성 구간은 "**branch 결정 시퀀스 일치** + 저장항 <1e-3 mm, Tsurf <1e-2 °C" 같은 **동작 일치** 기준으로. 비트 일치가 아니라 **결정경계 일치**를 검증 지표로.

### C6. 포팅 기준(Python)이 "LLM 변환본"이라는 신뢰 문제
`RoadSurf-Python/main.py` 헤더가 스스로 "LLM으로 Fortran에서 변환, 일부 차이 존재"라고 밝힌다. 이미 **지층 깊이식 불일치**(§8.1)를 발견했다. 즉 **검증 기준 자체가 버그를 포함**할 수 있다.
- **대응**: "1차 기준=Python"을 유지하되, **핵심 물리(식 27–32, 46–50)는 Fortran 원본과 직접 대조**를 M1–M2 필수 게이트로. 가능하면 Fortran을 컴파일해 예제 입력으로 **삼자 대조**(Fortran vs Python vs dROAD). 불일치 발견 시 Fortran을 진실로.

## Major — 설계 결함

### M1. BLC 반복 솔버: unroll 우선은 순서가 거꾸로
계획서는 BLC(최대 40회 반복)를 "1단계 unroll"로 시작한다고 한다. 그러나 BLC 반복은 **시간 루프(scan) 안에서 매 스텝** 돈다. 40×를 매 스텝 unroll + 역전파 저장 → **컴파일 시간·메모리 급증**, 게다가 원본은 1e-3에서 조기종료라 고정 40회는 결과도 다르다.
- **대응**: 처음부터 **암시적 미분 고정점**(`custom_vjp` 또는 jaxopt fixed-point)으로. 순전파는 조기종료 허용, 역전파는 수렴점 IFT 1회. §6의 "unroll 먼저" 문구 수정.

### M2. 방안 B(커플링 IFT)는 적용 전제가 안 맞을 수 있음
커플링은 매끄러운 고정점이 아니라 **bisection/secant + 25회 상한 + 실패 플래그 + 되감기**의 이산 알고리즘이다. IFT는 미분가능 잔차와 비영 야코비안을 요구 — 클램프·플래그가 낀 잔차엔 그대로 적용 불가.
- **대응**: 방안 B를 "IFT"라 단정하지 말고, **매끄러운 잔차 $r(c)=T_s^{end}(c)-T_{obs}$로 재정의한 뒤** 그 위에서만 IFT. 원본 알고리즘 재현이 목적이면 방안 B가 아니라 방안 C(제한적)임을 명시. 애초에 목적상 **방안 A로 충분**하다는 점을 더 강하게.

### M3. 매끄러운 게이트의 질량·에너지 보존 미보장
soft 게이트로 "일부는 눈, 일부는 물"로 분배하면 저수지 간 총량이 보존되지 않거나 비물리적 음수/과포화가 생길 수 있다. 계획서는 clip으로 음수만 막을 뿐 **보존 법칙을 설계 원칙으로 명시하지 않는다.**
- **대응**: "smooth 모드도 **질량보존·비음수**를 하드 제약으로 유지"를 §4 설계원칙에 추가. 전달량 = soft게이트 × (가용량)으로 정식화해 상한을 구조적으로 보장. 매 스텝 총수분 잔차를 진단 출력.

### M4. 동적 이산 상태를 "사전계산"으로 처리 불가
계획서 §4.4는 bool 플래그를 "마스킹/사전계산"한다지만 `VeryCold`, `WetSnowFrozen`, `SnowType`, `WearSurf` 등은 **진화하는 상태에 의존**해 사전계산 불가.
- **대응**: 이들을 **연속 상태**로 승격(예: VeryCold→sigmoid(지중온도)), 또는 straight-through로 처리하고 gradient 경로를 명시. §4.4를 "정적 이산(사전계산) vs 동적 이산(연속완화)"으로 이분.

### M5. float64 + GPU/vmap 아망의 모순
계획서 §11은 GPU 배치를 성능 목표 후보로 둔다. 그러나 명시적 stiff 스킴엔 float64가 사실상 필수인데, **소비자 GPU의 float64는 1/32~1/64 처리량**, TPU는 미지원. "float64 + GPU 배치"는 상충.
- **대응**: 성능 전략을 명확히 — CPU float64 + `vmap`(다지점) 우선, GPU는 float32 견딜 수 있는 부분(예: 매끄러운 대체 후 저stiff 구성)만. 혼합정밀도 여부를 조기 결정.

### M6. 일정이 앞은 쉽고 뒤는 과소평가
M1–M3(포팅·JAX화)는 현실적이나, **M4(미분가능화+smooth) 1주는 비현실적**. 질량보존 게이트, 엔탈피법 튜닝, 소실 gradient 대응, τ-annealing 스케줄은 프로젝트에서 **가장 연구적인 난제**다.
- **대응**: M4를 2–3주로, "gradient 병리 디버깅" 버퍼를 별도 배정. 총 일정 6–7주 → **9–11주**로 현실화. 각 마일스톤에 정량 exit 기준.

## Minor — 다듬기

- **m1. 순차 임계값 = 소실 gradient**: 물→눈→얼음→서리 사슬에 sigmoid를 다단 합성하면 도함수 곱이 0으로 수렴. 게이트 온도 τ를 넉넉히·스케일 정규화, 또는 log-space.
- **m2. scan은 되감기·조기종료 불가**: `simulation_failed` 조기탈출과 커플링 되감기는 고정길이 scan과 근본적으로 상충 — 방안 A로 되감기를 버린다는 점을 §4에 명시.
- **m3. 출력 노면온도 = 첫 두 층 평균**: 손실을 최상층이 아닌 **평균층**에 걸어야 관측과 정합(논문 3.2). loss 정의에 반영 확인.
- **m4. 태양위치(식 3–8, Meeus)·율리우스일**: `datetime.now()` 의존 제거하고 입력 시각 기반 결정론적 사전계산. SVF 경로 쓰면 "직달=0" 판정이 이산 — W 처리.
- **m5. 관측 결측 −9999 마스크**가 손실뿐 아니라 **초기화**(TsurfOBS로 1–4층 설정)에도 영향 — DA에서 초기상태를 제어변수로 두면 이 초기화 로직과 충돌 가능. 정리 필요.
- **m6. 라이브러리 리스크**: `flax.struct` 대신 순수 `NamedTuple`/`chex.dataclass`도 검토(의존 최소화). jaxopt 유지보수 상태 확인(일부 기능 optax로 이관 중).

## 공정하게 — 계획서의 강점
- 상태/파라미터/강제력 분리와 pytree·`lax.scan` 구조는 정석.
- exact/smooth **이중 모드**는 검증과 최적화를 분리하는 좋은 장치(단, C5·C4의 "무엇에 최적화하나" 문제는 남음).
- 논문 식별·**논문↔코드 불일치표**(§8.1)와 부록 B 대응표는 실무적으로 매우 유용.
- 커플링 방안 A/B/C 분해와 "A 우선"은 올바른 우선순위.
